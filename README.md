# คู่มือติดตั้งระบบจัดการแปลงที่ดิน (LTAX WebGIS) ให้ อบต. ใหม่

ใช้ checklist นี้ทุกครั้งที่ต้องนำระบบไปติดตั้งให้ อบต. แห่งใหม่ เพื่อไม่ให้พลาดขั้นตอนสำคัญ

---

## ภาพรวม

ระบบนี้ออกแบบเป็น **template** — 1 อบต. = 1 ชุดข้อมูล + 1 Netlify Site แยกกันเด็ดขาด ไม่ปนกัน ทำให้ปลอดภัยและดูแลง่าย

---

## Checklist ก่อนเริ่ม

- [ ] มีไฟล์ข้อมูลแปลงที่ดินของ อบต. ใหม่แล้ว (นามสกุล `.geojson`)
- [ ] รู้พิกัดศูนย์กลางพื้นที่ อบต. ใหม่ (lat, long)
- [ ] มีบัญชี GitHub และ Netlify พร้อมใช้งาน

---

## ขั้นตอนที่ 1: เตรียมไฟล์ข้อมูลแผนที่

คัดลอกโครงสร้างโปรเจกต์เดิมทั้งหมด แล้วแทนที่ไฟล์ข้อมูลในโฟลเดอร์ `data/` ด้วยข้อมูลของ อบต. ใหม่ ให้ตรงชื่อไฟล์ดังนี้:

| ชื่อไฟล์ | เลเยอร์ |
|---|---|
| `parcel.geojson` | แปลงที่ดินหลัก (Parcel) |
| `block.geojson` | Block |
| `zone.geojson` | Zone |
| `boundary.geojson` | Boundary |
| `building.geojson` | Building |
| `spk.geojson` | SPK |
| `แปลงเกษตร.geojson` | แปลงเกษตร |
| `แปลงชุมชน.geojson` | แปลงชุมชน |
| `boundarypoint.geojson` | Boundary Point |

> **หมายเหตุ:** ถ้า อบต. ใหม่ไม่มีข้อมูลบางเลเยอร์ ไม่ต้องกังวล ระบบถูกออกแบบให้ข้ามเลเยอร์ที่โหลดไม่ได้โดยอัตโนมัติ ไม่ error

---

## ขั้นตอนที่ 2: แก้ไข `js/config.js`

เปลี่ยนพิกัดศูนย์กลางแผนที่ให้ตรงกับพื้นที่ อบต. ใหม่:

```javascript
const START_CENTER = [14.6005, 104.2500];  // <- เปลี่ยนเป็น [lat, long] ของ อบต. ใหม่
const START_ZOOM   = 15;
```

**วิธีหาพิกัด:** เปิด Google Maps → คลิกขวาตรงกลางพื้นที่ อบต. → คัดลอกตัวเลขที่ขึ้นมา (เช่น `14.9285, 103.1234`)

---

## ขั้นตอนที่ 3: แก้ไข `index.html`

เปลี่ยนชื่อหัวเว็บให้ตรงกับ อบต. เพื่อไม่ให้สับสนเวลาเปิดหลายแท็บพร้อมกัน:

```html
<title>ระบบจัดการแปลงที่ดิน อบต.[ชื่อ อบต.]</title>
```

---

## ขั้นตอนที่ 4: สร้าง GitHub Repo ใหม่

1. สร้าง repository ใหม่บน GitHub เช่น `ltaxmap-[ชื่อ อบต.]`
2. อัปโหลดไฟล์ทั้งโปรเจกต์เข้าไป (ให้ไฟล์ `index.html`, `package.json` อยู่ที่ **root** ของ repo ไม่ซ้อนในโฟลเดอร์ย่อย)

โครงสร้างที่ถูกต้อง:
```
ltaxmap-[ชื่อ อบต.]/
  ├── css/
  ├── data/
  ├── js/
  ├── netlify/functions/
  │     ├── save.js      (ใช้ไฟล์เดิม ไม่ต้องแก้)
  │     └── load.js      (ใช้ไฟล์เดิม ไม่ต้องแก้)
  ├── index.html
  └── package.json       (ใช้ไฟล์เดิม ไม่ต้องแก้)
```

---

## ขั้นตอนที่ 5: Deploy บน Netlify

1. Netlify → **Add new site → Import an existing project → Deploy with GitHub**
2. เลือก repo ที่สร้างไว้
3. ตั้งค่า:

| ช่อง | ค่าที่ใส่ |
|---|---|
| Branch to deploy | `main` |
| Base directory | เว้นว่าง |
| Build command | เว้นว่าง |
| Publish directory | เว้นว่าง |
| Functions directory | `netlify/functions` (ค่า default อยู่แล้ว) |

4. กด **Deploy**

> **หมายเหตุ:** `save.js` / `load.js` ใช้โค้ดเดิมได้เลยโดยไม่ต้องแก้ เพราะ Netlify Blobs จะแยกที่เก็บข้อมูล (store) ตาม site โดยอัตโนมัติ — แต่ละ อบต. มี site เป็นของตัวเอง ข้อมูลจึงไม่มีทางปนกัน

---

## ขั้นตอนที่ 6: ทดสอบก่อนส่งมอบงานจริง

ทดสอบให้ครบทุกฟังก์ชันก่อนให้ อบต. ใช้งานจริง:

- [ ] แผนที่โหลดขึ้น พร้อมแปลงที่ดินของ อบต. ใหม่
- [ ] ปุ่ม **วาด** ใช้งานได้
- [ ] ปุ่ม **แก้ไข** ใช้งานได้
- [ ] ปุ่ม **แบ่ง** ใช้งานได้
- [ ] ปุ่ม **รวม** ใช้งานได้
- [ ] ปุ่ม **ลบ (Reset)** ใช้งานได้
- [ ] ปุ่ม **บันทึก** ขึ้น "✅ บันทึกเรียบร้อย"
- [ ] ปิดเว็บแล้วเปิดใหม่ → ข้อมูลที่บันทึกไว้ยังอยู่ครบ
- [ ] ปุ่ม **ค้นหา** ใช้งานได้
- [ ] ปุ่ม **Import / Export** ใช้งานได้

### การเพิ่ม อบต แก้ไขใน worker,layer,storage
#_worker.js → เพิ่มรหัส อบต. ต่อท้ายในลิสต์ ALLOWED_ABT (และเปลี่ยน PARCEL_KV เป็น KV_BINDING ให้ตรงกับ binding ของ project map)
#storage.js → เติม ?abt=<รหัสอบต.นั้น> ต่อท้าย /api/save
#layers.js → เติม ?abt=<รหัสอบต.นั้น> ต่อท้าย /api/load

##ตัวอย่าง
#layer# // const savedRes = await fetch('/api/load?abt=lalom', { cache: "no-store" });
##
#storage#// เรียก Cloudflare Worker (ต้องระบุ abt เสมอ ไม่งั้น worker จะปฏิเสธด้วย 400)
  const saveUrl = '/api/save?abt=lalom';

*(เพิ่มแถวใหม่ทุกครั้งที่ติดตั้งให้ อบต. เพิ่ม)*
