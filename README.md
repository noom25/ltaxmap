## 1. ภาพรวมระบบทำงานยังไง

ระบบนี้ทำงานบน **Cloudflare Workers** ซึ่งรวม 3 ส่วนไว้ในที่เดียว:

```
ผู้ใช้เปิดเว็บ
      │
      ▼
┌─────────────────────────────────────┐
│         Cloudflare Worker            │
│                                       │
│  ถ้า URL เป็น /api/save, /api/load   │──▶  รันโค้ดใน _worker.js
│  ถ้าเป็น URL อื่นๆ                    │──▶  ส่งไฟล์ static (index.html, css, js)
│                                       │
└─────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────┐
│   Workers KV (PARCEL_KV)             │  ← ที่เก็บข้อมูล geojson ที่กด Save ไว้
│   - parcel.geojson (ข้อมูลล่าสุด)     │
│   - backup-YYYY-MM-DD.geojson         │  ← backup รายวัน เก็บย้อนหลัง 30 วัน
└─────────────────────────────────────┘
```

**สรุปสั้นๆ:**
- หน้าเว็บ (แผนที่) = ไฟล์ static ธรรมดา (HTML/CSS/JS)
- ปุ่ม "Save" บนเว็บ = ยิงไปที่ `/api/save` ให้ Worker บันทึกลง KV
- ตอนโหลดหน้าเว็บ = ดึงข้อมูลล่าสุดจาก `/api/load`
- ทั้งหมดนี้ deploy ผ่าน **GitHub** โดยไม่ต้องลงโปรแกรมอะไรในเครื่องเลย

## 2. โครงสร้างโฟลเดอร์ทั้งหมด

```
ltaxmap/                          ← root ของ repo (ทุกไฟล์ต้องอยู่ระดับนี้หรือลึกลงไป)
│
├── _worker.js                    ← โค้ด backend (save/load ผ่าน KV)
├── wrangler.jsonc                ← config บอก Cloudflare ว่า worker/KV ผูกกันยังไง
├── .assetsignore                 ← กัน _worker.js หลุดไปเป็น static asset
├── index.html                    ← หน้าเว็บหลัก (ตัวแผนที่)
├── package.json                  ← รายชื่อ dependency ของโปรเจกต์
│
├── css/                          ← ไฟล์สไตล์หน้าเว็บ
│
├── js/                           ← โค้ด frontend
│   ├── layers.js                 ← จัดการเลเยอร์แผนที่ (parcel, block, zone ฯลฯ)
│   └── storage.js                ← ปุ่ม save/export/import ข้อมูล
│
├── data/                         ← ไฟล์ GeoJSON ตั้งต้น (ข้อมูลเริ่มต้นก่อนมีคนแก้ไข)
│   └── parcel.geojson, block.geojson, zone.geojson ฯลฯ
│
├── doc/                          ← เอกสารประกอบ (ไม่เกี่ยวกับการรันเว็บ)
│
└── netlify/functions/            ← ⚠️ ของเก่าจากตอนใช้ Netlify ไม่ได้ใช้แล้ว
                                      (เก็บไว้ก็ได้ ไม่กระทบการทำงาน หรือจะลบทิ้งก็ได้)
```

## 3. ไฟล์ไหนสำคัญ ไฟล์ไหนไม่ต้องแตะ

| ไฟล์/โฟลเดอร์ | หน้าที่ | ต้องแก้ตอนทำ อบต. ใหม่ไหม |
|---|---|---|
| `_worker.js` | โค้ด backend save/load | ❌ ไม่ต้องแก้ (เหมือนกันทุกที่) |
| `.assetsignore` | กันไฟล์ worker หลุดเป็น static | ❌ ไม่ต้องแก้ (เหมือนกันทุกที่) |
| `wrangler.jsonc` | ผูก worker + KV | ✅ **ต้องแก้ 2 จุดทุกครั้ง** (ดูข้อ 5) |
| `index.html`, `css/`, `js/` | หน้าตาเว็บ | ✅ แก้ตามความต้องการของแต่ละ อบต. (ชื่อ, โลโก้ ฯลฯ) |
| `data/*.geojson` | ข้อมูลแผนที่ตั้งต้น | ✅ **ต้องเปลี่ยนเป็นข้อมูลของ อบต. นั้นๆ** |
| `netlify/functions/` | ของเก่า ไม่ได้ใช้ | ลบทิ้งได้ ไม่กระทบอะไร |

## 4. วิธี deploy ครั้งแรก

ไม่ต้องลง Node.js หรือ Wrangler CLI เลย ทำผ่านหน้าเว็บทั้งหมด:

### ขั้นตอน
1. อัปโหลดไฟล์ทั้งหมดขึ้น **GitHub repository**
2. เข้า **Cloudflare Dashboard** → **Workers & Pages** → **Create**
3. เลือก **Connect to Git** → เลือก repo ที่เพิ่ง push ขึ้นไป
4. ตั้งค่า Build:
   - Build command: เว้นว่างไว้ (ไม่ต้องมี)
   - Deploy command: `npx wrangler deploy`
5. กด **Save and Deploy**
6. รอ build เสร็จ แล้วเข้าไปเช็คที่แท็บ **Bindings** ว่ามี:
   - `Assets` → `ASSETS`
   - `KV namespace` → `PARCEL_KV` → ชื่อ namespace ที่สร้างไว้

ถ้าเห็นครบ 2 อันนี้ = ใช้งานได้แล้ว กดปุ่ม **Visit** เพื่อเปิดเว็บทดสอบได้เลย

### หลังจากนั้น (แก้โค้ด/ข้อมูล)
ถ้า **เชื่อมต่อ Git ไว้** (ไม่ได้กด Disconnect) แค่แก้ไฟล์บน GitHub แล้ว commit → Cloudflare จะ deploy ให้อัตโนมัติ ไม่ต้องทำอะไรเพิ่ม

ถ้า **เคย Disconnect ไปแล้ว** ต้องเข้าไปกด **Settings → Build → Connect** เพื่อเชื่อมกลับ ไม่งั้นแก้โค้ดแล้วจะไม่ deploy ให้เอง ต้องกด **New deployment** เองทุกครั้ง

## 5. วิธีทำระบบนี้ให้ อบต. อื่น

ใช้ checklist นี้ทุกครั้งที่ทำ อบต. ใหม่ ทำตามลำดับ ห้ามข้ามข้อ

### ขั้นที่ 1 — เตรียมข้อมูล อบต. ใหม่
คัดลอกโฟลเดอร์โปรเจกต์เดิมทั้งหมดไปเป็นชุดใหม่ แล้วแก้:
- `index.html` — เปลี่ยนชื่อ อบต., โลโก้, ข้อความในหน้าเว็บ
- `data/*.geojson` — เปลี่ยนเป็นข้อมูลแปลงที่ดิน/เขต ของ อบต. นั้น

### ขั้นที่ 2 — สร้าง KV Namespace ใหม่ (ห้ามใช้ตัวเดิมซ้ำ)
1. เข้า Cloudflare Dashboard → ค้นหา **"KV"** ที่ Quick search
2. กด **Create namespace**
3. ตั้งชื่อให้รู้ว่าเป็นของที่ไหน เช่น `abt-นครชัยศรี-kv`
4. กด Create แล้วก็อป **Namespace ID** เก็บไว้

> ⚠️ **สำคัญมาก:** ถ้าลืมสร้างใหม่แล้วเผลอใช้ id เดิมซ้ำ ข้อมูลของ อบต. ต่างกันจะไปเก็บอยู่ใน KV เดียวกัน แล้วทับ/ปนกันได้ทันที

### ขั้นที่ 3 — แก้ไฟล์ `wrangler.jsonc`
เปิดไฟล์ (copy จาก template) แก้ 2 จุด:

```jsonc
{
  "$schema": "node_modules/wrangler/config-schema.json",

  // 🔧 จุดที่ 1: เปลี่ยนชื่อนี้ให้ไม่ซ้ำกับ Worker อื่น
  "name": "map-นครชัยศรี",

  "main": "_worker.js",
  "compatibility_date": "2026-07-16",
  "compatibility_flags": ["nodejs_compat"],
  "assets": {
    "directory": ".",
    "binding": "ASSETS"
  },
  "kv_namespaces": [
    {
      "binding": "PARCEL_KV",
      // 🔧 จุดที่ 2: ใส่ Namespace ID ใหม่จากขั้นที่ 2
      "id": "ใส่-namespace-id-ที่สร้างใหม่ตรงนี้"
    }
  ],
  "observability": { "enabled": true }
}
```

### ขั้นที่ 4 — วางไฟล์ครบชุดที่ root ของ repo ใหม่
ต้องมีครบ: `_worker.js`, `.assetsignore`, `wrangler.jsonc` (ที่แก้แล้ว), `index.html`, `css/`, `js/`, `data/`

### ขั้นที่ 5 — Push ขึ้น GitHub แล้วเชื่อม Cloudflare
1. สร้าง repo ใหม่ใน GitHub (แนะนำให้แยก repo ต่อ อบต. เพื่อไม่ให้ปนกัน)
2. Push ไฟล์ทั้งหมดขึ้นไป
3. Cloudflare Dashboard → Workers & Pages → Create → เชื่อมต่อกับ repo ใหม่นี้
4. รอ build เสร็จ

### ขั้นที่ 6 — ทดสอบ
1. กดปุ่ม Visit เปิดเว็บ
2. ลองวาด/แก้ข้อมูลแล้วกด Save
3. Refresh หน้าเว็บ เช็คว่าข้อมูลที่บันทึกโหลดกลับมาได้
4. เข้าแท็บ Bindings เช็คว่ามี `ASSETS` + `PARCEL_KV` (ชื่อ namespace ของ อบต. นี้) ครบ

## 6. วิธีโหลดข้อมูลทั้งหมดกลับมาเก็บ

แบ่งเป็น 2 ส่วน เพราะเก็บคนละที่กัน:

### ก) โค้ดเว็บทั้งหมด (จาก GitHub)
1. เข้าหน้า repo บน GitHub
2. กดปุ่มสีเขียว **`<> Code`** → **Download ZIP**
3. ได้ไฟล์ทั้งหมดของเว็บทันที (ไม่ต้องลงอะไรเพิ่ม)

### ข) ข้อมูลแปลงที่ดินที่เคย Save ไว้ (จาก Cloudflare KV)
ข้อมูลนี้**ไม่ได้อยู่ใน GitHub** ต้องดึงจาก KV โดยตรงผ่านหน้า Dashboard (ไม่ต้องลง Node.js):

1. เข้า Cloudflare Dashboard → **Storage & Databases → KV**
2. เลือก namespace ของ อบต. นั้น (เช่น `ltaxmap-data`)
3. จะเห็นรายการ key เช่น `parcel.geojson`, `backup-2026-07-16.geojson` ฯลฯ
4. กดเข้าไปที่ key แต่ละอัน → เนื้อหา JSON จะแสดงขึ้นมาให้ copy
5. Copy แล้ววางใน Notepad แล้วเซฟเป็นไฟล์ `.geojson` เก็บไว้เป็น backup

> 💡 แนะนำให้ backup ข้อมูลแบบนี้เป็นระยะ โดยเฉพาะก่อนช่วงที่จะมีการแก้ไขแผนที่ครั้งใหญ่

## 7. ปัญหาที่เคยเจอ และวิธีแก้

### ปัญหา: กดปุ่ม "Add binding" แล้วไม่มีอะไรเกิดขึ้น
**สาเหตุ:** Worker อยู่ในสถานะ "static assets only" เพราะ deploy fail ทุกครั้ง จึงไม่มีโค้ด Worker จริงให้ผูก binding ได้

**วิธีแก้:** ไม่ต้องพึ่งปุ่มนี้เลย เขียน binding ลงใน `wrangler.jsonc` โดยตรงแทน (ตามขั้นตอนข้อ 4-5 ด้านบน) แล้ว deploy ผ่าน Git — Cloudflare จะสร้าง binding ให้อัตโนมัติตาม config

### ปัญหา: Deploy fail ด้วย error `Uploading a Pages _worker.js file as an asset`
**สาเหตุ:** ระบบ auto-detect โปรเจกต์เป็น "Static" (ไม่รู้ว่า `_worker.js` คือโค้ดที่ต้องรัน) เลยพยายามอัปโหลดเป็นไฟล์ธรรมดาแทน ซึ่งอันตราย (โค้ด backend จะถูกเปิดเผยให้คนอื่นดาวน์โหลดได้)

**วิธีแก้:** ต้องมีไฟล์ **`.assetsignore`** ที่ root ของ repo ใส่บรรทัด `_worker.js` และมี **`"main": "_worker.js"`** ใน `wrangler.jsonc` ทั้งสองอย่างต้องมีครบ ถึงจะ deploy ผ่าน

### ปัญหา: สร้าง/แก้ไฟล์ dotfile (เช่น `.assetsignore`) แล้วดูเหมือนไม่ถูก push ขึ้น GitHub
**สาเหตุ:** การอัปโหลดไฟล์แบบลากวาง (drag & drop) ผ่านหน้าเว็บ บางครั้งเบราว์เซอร์ไม่ยอมรับไฟล์ที่ขึ้นต้นด้วยจุด

**วิธีแก้ที่ชัวร์สุด:** เข้า GitHub → **Add file → Create new file** → พิมพ์ชื่อไฟล์ `.assetsignore` ตรงๆ ในช่องชื่อไฟล์ → ใส่เนื้อหา → Commit

### ปัญหา: Deploy ผ่านแล้ว แต่ auto-deploy ไม่ทำงานอีกเมื่อแก้โค้ด
**สาเหตุ:** เคยกด **Disconnect** repo ออกจาก Worker (อาจกดโดยไม่ตั้งใจตอนดูหน้า Settings)

**วิธีแก้:** เข้า Worker → **Settings → Build → Connect** แล้วเชื่อมต่อ repo เดิมกลับเข้าไปใหม่ ไม่มีผลเสียใดๆ เว็บที่ deploy ไปแล้วยังทำงานปกติระหว่างที่ยังไม่ได้เชื่อมกลับ

## 8. คำถามที่พบบ่อย

**Q: ต้องลง Node.js ไหม ถ้าไม่อยากยุ่งกับ command line?**
A: ไม่ต้องครับ ถ้า deploy ผ่าน GitHub (Git integration) ทำทุกอย่างผ่านหน้าเว็บได้หมด รวมถึงดูข้อมูล KV ก็ดูผ่าน Dashboard ได้โดยไม่ต้องมี Node.js เลย Node.js/Wrangler CLI จำเป็นเฉพาะตอนอยาก deploy จากเครื่องตัวเองโดยตรง หรืออยากดึงข้อมูล KV แบบ bulk ผ่านคำสั่งเดียว

**Q: deploy ผ่าน Git บ่อยๆ จะเสียเงินไหม?**
A: แผน Free ให้ 3,000 build minutes ต่อเดือน งานที่ deploy ไม่บ่อย (เป็นบางช่วง ไม่ใช่ทุกวัน) แทบไม่มีทางแตะโควตานี้เลย ปกติแทบไม่มีค่าใช้จ่ายอะไร

**Q: ทำหลาย อบต. ต้องแยก repo กันไหม?**
A: แนะนำให้แยก repo ต่อ อบต. เพื่อไม่ให้โค้ด/ข้อมูลปนกัน และง่ายต่อการดูแลแยกส่วน แต่ละ repo ก็สร้าง Worker + KV namespace แยกกันตามขั้นตอนในข้อ 5

**Q: ถ้าอยากเปลี่ยนชื่อโดเมนของแต่ละ อบต. ทำยังไง?**
A: เข้า Worker นั้น → แท็บ **Domains** → เพิ่ม custom domain หรือใช้ subdomain ฟรีที่ Cloudflare ให้ (รูปแบบ `ชื่อ-worker.บัญชี.workers.dev`)
